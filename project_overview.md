# 前沿技术情报与预警平台 V1.5

## 一、项目目标

构建一个结构化的“前沿技术情报与预警平台”，实现以下核心能力：

1. 情报上传与分类
2. 自动指标抽取（基于规则）
3. 人工修正指标
4. 预警生成与人工调整
5. 情报发布与状态管理
6. 多维度筛选与统计展示
7. 趋势分析与汇总

目标是建立一个“上传 → AI建议 → 人工修正 → 发布 → 筛选/统计”的完整闭环。

## 三、当前数据存储方式

- 基础数据：data/*.json
- 新发布情报：localStorage
- 手动预警：localStorage

当前为“前端样机阶段”，未接后端。

---

## 四、核心业务流程（目标闭环）

1. 用户填写情报信息
2. 点击“提交并生成AI建议”
3. 系统根据正文抽取指标建议
4. 用户可修改指标名、数值、单位
5. 可选择是否生成预警（若生成必须填写原因）
6. 点击发布
7. 情报进入总表
8. 参与筛选与统计

---

## 五、设计原则

1. 前端与未来后端数据结构保持一致
2. 所有字段未来可直接映射数据库
3. AI只做建议，不自动写入
4. 发布前必须经过人工确认
5. 多领域支持（含父子域结构）
6. 所有筛选基于 published 状态
