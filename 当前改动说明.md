# 当前改动说明（2026-03-03）

## 1. 目标
将情报主链路从前端 `localStorage` 迁移到后端 `FastAPI + PostgreSQL`：
- 发布写入数据库
- 总表读取数据库
- 详情读取数据库
- 总表删除写数据库

## 2. 后端改动

### 2.1 数据库与迁移
- 新增 SQLAlchemy 数据层：
  - `backend/app/db/base.py`
  - `backend/app/db/session.py`
- 新增模型：
  - `backend/app/models/domain.py`
  - `backend/app/models/info_item.py`
- 新增 Alembic：
  - `backend/alembic.ini`
  - `backend/alembic/env.py`
  - `backend/alembic/versions/0001_init_tables.py`

### 2.2 API 接口
- `GET /api/info-items?status=published`：读取数据库中的情报列表
- `POST /api/info-items/publish`：发布并写入数据库
- `GET /api/info-items/{id}`：读取单条情报详情
- `DELETE /api/info-items/{id}`：删除单条情报

### 2.3 启动与配置
- `backend/Dockerfile` 启动时执行：`alembic upgrade head`
- `docker-compose.yml` 新增 `db`（PostgreSQL）服务
- API 环境变量新增：
  - `DATABASE_URL`
  - `DATA_DIR=/app/data`
- 启动阶段会自动尝试初始化 domains（数据库为空时从 `data/domains.json` 导入）

### 2.4 依赖
`backend/requirements.txt` 新增：
- `sqlalchemy`
- `psycopg2-binary`
- `alembic`
- `httpx`

## 3. 前端改动

### 3.1 upload_review.html
- 发布动作改为调用 `POST /api/info-items/publish`
- 发布成功后跳转详情页：`info_detail.html?id=<uuid>`
- 领域数据改为调用 `GET /api/domains/tree`
- 保留本地草稿能力（`drafts`）

### 3.2 info_table.html
- 列表改为调用 `GET /api/info-items?status=published`
- 删除按钮改为调用 `DELETE /api/info-items/{id}`
- 不再依赖 `published_items` / `deleted_info_ids` 本地存储

### 3.3 info_detail.html
- 详情数据改为调用 `GET /api/info-items/{id}`
- 仍保留指标字典/指标值静态展示逻辑（样机阶段）

### 6.4 批量导入增强（草稿 + 直接发布）

### 7.8 本次补充修复与验证（2026-03-08，晚间）

#### 7.8.1 修复：批量“直接发布”500错误
- 问题：`POST /api/info-items/publish` 在“非 draft_id 直接发布”路径中，先 `flush` 再赋值 `title/content`，触发数据库非空约束报错。
- 现象：接口返回 500，日志为 `null value in column "title" of relation "info_items" violates not-null constraint`。
- 修复文件：
  - `backend/app/routers/info_items.py`
- 修复方式：
  - 新建发布对象后先应用 `_apply_upsert_fields(item, payload)`，再进行首次 `flush`。

#### 7.8.2 增强：Excel 批量上传宽容性
- 修复文件：
  - `excel_import.html`
- 关键改动：
  - `.xlsx` 日期增强：兼容 Excel 序列日期、`yyyy/m/d`、`yyyy-mm-dd`，统一归一化后再提交。
  - 列表分隔增强：`source_types/info_types/domains/tags` 支持 `;`、`；`、`,`、`，`。
  - 布尔兼容增强：支持 `true/1/yes/y/是`。
  - 校验策略放宽：允许“指标 name 有值但 value 为空”进入批量草稿。
  - 批量失败提示增强：弹窗返回前几条具体后端错误信息，便于定位。
