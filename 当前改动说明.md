# 当前改动说明（2026-03-03）

## 1. 目标
将情报主链路从前端 `localStorage` 迁移到后端 `FastAPI + PostgreSQL`：
- 发布写入数据库
- 总表读取数据库
- 详情读取数据库
- 总表删除写数据库

## 2. 后端改动

### 2.1 数据库与迁移
- 新增 SQLAlchemy 数据层：
  - `backend/app/db/base.py`
  - `backend/app/db/session.py`
- 新增模型：
  - `backend/app/models/domain.py`
  - `backend/app/models/info_item.py`
- 新增 Alembic：
  - `backend/alembic.ini`
  - `backend/alembic/env.py`
  - `backend/alembic/versions/0001_init_tables.py`

### 2.2 API 接口
- `GET /api/info-items?status=published`：读取数据库中的情报列表
- `POST /api/info-items/publish`：发布并写入数据库
- `GET /api/info-items/{id}`：读取单条情报详情
- `DELETE /api/info-items/{id}`：删除单条情报

### 2.3 启动与配置
- `backend/Dockerfile` 启动时执行：`alembic upgrade head`
- `docker-compose.yml` 新增 `db`（PostgreSQL）服务
- API 环境变量新增：
  - `DATABASE_URL`
  - `DATA_DIR=/app/data`
- 启动阶段会自动尝试初始化 domains（数据库为空时从 `data/domains.json` 导入）

### 2.4 依赖
`backend/requirements.txt` 新增：
- `sqlalchemy`
- `psycopg2-binary`
- `alembic`
- `httpx`

## 3. 前端改动

### 3.1 upload_review.html
- 发布动作改为调用 `POST /api/info-items/publish`
- 发布成功后跳转详情页：`info_detail.html?id=<uuid>`
- 领域数据改为调用 `GET /api/domains/tree`
- 保留本地草稿能力（`drafts`）

### 3.2 info_table.html
- 列表改为调用 `GET /api/info-items?status=published`
- 删除按钮改为调用 `DELETE /api/info-items/{id}`
- 不再依赖 `published_items` / `deleted_info_ids` 本地存储

### 3.3 info_detail.html
- 详情数据改为调用 `GET /api/info-items/{id}`
- 仍保留指标字典/指标值静态展示逻辑（样机阶段）

## 6. 本次增量改动（2026-03-03，追加）

### 6.1 首页工作台数据源调整
- `index.html` 不再读取 `data/info_items.json + localStorage published_items` 预设合并数据。
- 改为统一调用后端：
  - `GET /api/domains/tree`
  - `GET /api/info-items?status=published`
- 首页仅展示新上传并已发布到数据库的数据。

### 6.2 预警后端化（颜色/到期时间/人工编辑）
- 后端 `info_items` 扩展预警字段（写在同表）：
  - `alert_status`
  - `alert_source`
  - `alert_manual_override`
  - `alert_title`
  - `alert_body`
  - `alert_reviewer_comment`
  - `alert_dismiss_reason`
  - `alert_color`
  - `alert_expires_at`
- 新增接口：
  - `PATCH /api/info-items/{id}/alert`（人工编辑/解除预警）
- `POST /api/info-items/publish` 支持 `alert` 入参，发布时即可写入预警与时限。
- 情报总表 `info_table.html`：
  - 恢复预警展示（颜色 + 到期日期）
  - 人工预警弹窗支持编辑到期时间并写后端
- 详情页 `info_detail.html`：
  - 删除“模拟：延期 / 模拟：解除”
  - 仅保留“人工编辑预警”，且可修改预警时间并写后端

### 6.3 上传/修订稿回流修复
- `upload_review.html`：
  - 发布时把 `duration` 转换为 `expires_at` 写后端，修复“时限不生效”
  - 草稿保存增加预警相关字段与指标，避免回填丢失
  - 从修订稿/草稿回填时恢复领域、密级、预警字段、指标
- `info_detail.html` 创建修订稿时，回流携带原始字段（含预警状态相关字段）。

### 6.4 批量导入增强（草稿 + 直接发布）
- `excel_import.html`：
  - 原“批量生成草稿”保留
  - 新增“批量发布（跳过草稿）”按钮，逐行调用发布接口
  - CSV 解析增强：支持引号/逗号场景
  - 中文编码兼容：优先 UTF-8，乱码时回退 `GB18030`，修复中文乱码

### 6.5 模板改造
- 新增模板文件：
  - `data/draft_template.csv`（草稿模板）
  - `data/upload_template.csv`（批量上传模板）
- 上传模板包含所有预警字段，并在末尾提供最多 20 组指标列：
  - `indicator_1_name/value/unit` ... `indicator_20_name/value/unit`

### 6.6 FastAPI 弃用警告修复
- `backend/app/main.py` 将 `@app.on_event("startup")` 改为 `lifespan` 事件处理。
- 对应警告来源已移除：
  - `on_event is deprecated, use lifespan event handlers instead`

### 6.7 数据库迁移
- 新增 Alembic 迁移：
  - `backend/alembic/versions/0002_add_alert_columns.py`
- 用于给已有数据库补齐预警字段。

### 6.8 本地验证情况
