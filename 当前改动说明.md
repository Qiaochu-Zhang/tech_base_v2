# 当前改动说明（2026-03-03）

## 1. 目标
将情报主链路从前端 `localStorage` 迁移到后端 `FastAPI + PostgreSQL`：
- 发布写入数据库
- 总表读取数据库
- 详情读取数据库
- 总表删除写数据库

## 2. 后端改动

### 2.1 数据库与迁移
- 新增 SQLAlchemy 数据层：
  - `backend/app/db/base.py`
  - `backend/app/db/session.py`
- 新增模型：
  - `backend/app/models/domain.py`
  - `backend/app/models/info_item.py`
- 新增 Alembic：
  - `backend/alembic.ini`
  - `backend/alembic/env.py`
  - `backend/alembic/versions/0001_init_tables.py`

### 2.2 API 接口
- `GET /api/info-items?status=published`：读取数据库中的情报列表
- `POST /api/info-items/publish`：发布并写入数据库
- `GET /api/info-items/{id}`：读取单条情报详情
- `DELETE /api/info-items/{id}`：删除单条情报

### 2.3 启动与配置
- `backend/Dockerfile` 启动时执行：`alembic upgrade head`
- `docker-compose.yml` 新增 `db`（PostgreSQL）服务
- API 环境变量新增：
  - `DATABASE_URL`
  - `DATA_DIR=/app/data`
- 启动阶段会自动尝试初始化 domains（数据库为空时从 `data/domains.json` 导入）

### 2.4 依赖
`backend/requirements.txt` 新增：
- `sqlalchemy`
- `psycopg2-binary`
- `alembic`
- `httpx`

## 3. 前端改动

### 3.1 upload_review.html
- 发布动作改为调用 `POST /api/info-items/publish`
- 发布成功后跳转详情页：`info_detail.html?id=<uuid>`
- 领域数据改为调用 `GET /api/domains/tree`
- 保留本地草稿能力（`drafts`）

### 3.2 info_table.html
- 列表改为调用 `GET /api/info-items?status=published`
- 删除按钮改为调用 `DELETE /api/info-items/{id}`
- 不再依赖 `published_items` / `deleted_info_ids` 本地存储

### 3.3 info_detail.html
- 详情数据改为调用 `GET /api/info-items/{id}`
- 仍保留指标字典/指标值静态展示逻辑（样机阶段）
